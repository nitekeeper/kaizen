# internal/open-pr

Bundled PR opening. Backed by `scripts/pr.py` (`open_pr_for_run`), which already encodes the full sequence: load run + project + cycles + abandonments → render title/body per design §4.6 → invoke `gh pr create` from inside the clone → persist the returned URL onto the runs row.

## Inputs

- `run_id` (int) — the run whose cycles are being bundled into the PR.
- `clone_dir` (Path) — the experiment clone where the push happened. Required because `gh pr create` runs from inside the repo.

## Output

The PR URL as a string. The same URL is also persisted onto `runs.pr_url`.

## Procedure

### Step 1 — Verify gh is authenticated

Already checked at the top of the run by `scripts/setup.py`, but cheap to re-confirm:

```
gh auth status
```

Exit 0 means good. Non-zero means re-auth needed — abort and surface the error.

### Step 2 — Invoke the open-PR helper

```
PYTHONPATH=. python3 scripts/pr.py <run_id> <clone_dir>
```

This single command:

1. Loads the run, the project, every cycle (ordered by `cycle_n`), and every abandonment (keyed by `cycle_id`).
2. Renders the PR title:
   ```
   kaizen: <subject or "PM-directed"> — N cycles, S succeeded / A abandoned
   ```
3. Renders the body per design §4.6 (Summary table → per-cycle sections → Abandonment reports section listing memex slugs).
4. Runs `gh pr create --title ... --body ... --base <project.base_branch> --head <run.branch>` from inside `clone_dir`.
5. Captures the printed URL (gh prints it as the last non-empty stdout line).
6. Writes the URL onto `runs.pr_url` via `update_run_pr_url`.

Stdout: the PR URL.

### Step 3 — Capture and return

Read the URL from stdout. Return it to the caller (`internal/run/SKILL.md`), which will pass it through `internal/run-record/SKILL.md` (`finalize`) and surface it to the user.

### Failure handling

- **`gh pr create` fails** (branch protection, base mismatch, network): `scripts/pr.py` exits non-zero and prints the gh stderr. Do not delete the clone; surface the error and signal failure to the orchestrator so `finalize_run` is called with `status='failed'`.
- **Empty stdout despite exit 0**: `pr.py` raises `RuntimeError("gh pr create returned exit 0 but no URL on stdout")`. Same handling — do not delete the clone, surface, fail.
- **All cycles abandoned (`cycles_succeeded == 0`)**: the PR is still opened. The body has no commit shas, just abandonment summaries. This is the intended design §3.3 behaviour — let the user see the reports.

## PR body shape (reference)

```markdown
## Summary

Multi-cycle improvement run against this repo.

| | |
|---|---|
| Cycles requested | N |
| Succeeded | S |
| Abandoned | A |
| Run started | YYYY-MM-DD HH:MM UTC |
| Run ended | YYYY-MM-DD HH:MM UTC |

## Cycles

### Cycle 1 — success
- Subject: <subject or PM-directed>
- Commit: `abc1234`
- Minutes (Kaizen wiki): `kaizen:cycle:<run-id>-1`

### Cycle 2 — abandoned
- Subject: ...
- Commit: —
- Phase reached: meeting
- Reason: no_consensus
- Report: `kaizen:abandonment:<run-id>-cycle-2`
- Detail summary: <first 200 chars...>

## Abandonment reports

See Kaizen memex entries:
- `kaizen:abandonment:<run-id>-cycle-2`

🤖 Generated by Kaizen against <project name> at <git_url>
```

`scripts/pr.py:render_pr_body` produces this exact shape; do not hand-render — call the helper.

## Hard rules

- **One PR per run.** Even when cycles_succeeded is 0, exactly one PR is opened. The PR may be commit-empty (no successes) but is never skipped.
- **Always invoke from inside `clone_dir`.** `gh pr create` requires a git repo cwd; the helper sets it explicitly.
- **Do not modify the rendered body before passing to gh.** The body is a published contract — manual edits drift it from the template the user expects.
- **The clone must survive until this step succeeds.** Teardown (`internal/clone-target/SKILL.md` `teardown`) happens AFTER the PR opens, in `internal/run/SKILL.md` step 10.
